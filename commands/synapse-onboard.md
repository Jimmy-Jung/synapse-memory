!`SYNAPSE_FROM_AGENT=1 synapse-memory assistant-status --json`

위 출력은 현재 vault 상태 스냅샷(JSON)입니다. **온보딩은 한 세션에 답답함 1개만 다룹니다** — 다 보여주려 하면 사용자가 압도됩니다.

다음 스크립트를 따라 사용자를 인도하세요. 각 단계에서 *질문 → 사용자 응답 → 다음 슬래시 호출* 흐름을 지키고, **위험한 작업은 사용자 동의 후 실행**합니다.

## 1단계 — 환경 점검 보고 (10초)

위 JSON의 `doctor_ok`, `vault_path`만 봅니다.

- `doctor_ok: false`이면: 사용자에게 알리고 `/synapse-fix`를 *제안만* 합니다 (자동 실행 금지).
- `vault_path: null`이면: Obsidian vault 경로 설정 안내 후 종료. 다음 단계 진행 불가.
- 둘 다 OK이면: "환경 준비 완료. 이제 시작해도 됩니다." 한 줄만 보고하고 다음 단계로.

## 2단계 — 답답함 선택 (사용자 응답 대기)

사용자에게 *정확히 이 형식*으로 질문하세요.

```
Synapse Memory가 풀어주는 5가지 답답함이 있습니다.
오늘은 1개만 골라서 끝까지 체험해보겠습니다. 어떤 것이 가장 와닿으세요?

1️⃣ "그 노트 어디 있더라?" — 노트가 쌓이는데 검색해도 안 나옴
2️⃣ "Claude에 매번 내 맥락을 처음부터 설명한다"
3️⃣ "AI에 개인 자료 던지자니 회사 NDA가 걱정"
4️⃣ "회사마다 이력서를 6시간씩 다시 쓴다"
5️⃣ "사소한 결정에 매일 피로하다"

번호로 답해주세요 (1~5). 잘 모르겠으면 "skip"이라고 적어도 됩니다.
```

사용자 응답을 기다리세요. 응답 전에 다음 단계로 넘어가지 마세요.

## 3단계 — 선택에 따라 분기 (한 가지만 끝까지)

### 1️⃣ 선택 → 검색 체험

1. "먼저 자료를 모으고 카드를 만듭니다. 1~3분 걸립니다." 안내
2. `/synapse-daily` 호출 (사용자 동의 후)
3. 결과에서 생성된 카드 수 보고
4. 새 카드 1개를 골라 시연: `/synapse-ask "<카드 주제>에 대해 정리해줘"`
5. 답변에 *출처 카드 ID*가 붙은 것을 짚어 보여주기

### 2️⃣ 선택 → 컨텍스트 자동 주입 체험

1. 사용자에게 자신이 *반복해서* Claude에 설명하던 주제 하나를 묻기 (예: "우리 팀 코딩 컨벤션", "내가 쓰는 기술 스택")
2. 카드가 없으면 `/synapse-daily` 한 번 실행해서 카드를 만들어두기
3. `/synapse-ask "<그 주제>"` 호출 → 답변에 자동으로 컨텍스트가 붙는 것 보여주기

### 3️⃣ 선택 → 보안 체험

1. "외부 AI에 절대 보내고 싶지 않은 회사명·프로젝트명이 있나요?" 묻기
2. 사용자 응답을 받아 `/synapse-ask "redact-list에 '<단어>' 추가해줘"` 호출
3. `synapse-memory redactlist show`로 등록 확인
4. "이제 모든 외부 AI 호출에서 이 단어는 `[REDACT_*]`로 치환됩니다" 설명
5. 마스킹 2단계 흐름 짧게 요약 ([for-everyone/how-it-works.md#답답함-3](../docs/for-everyone/how-it-works.md) 참고)

### 4️⃣ 선택 → 이력서 체험

1. 사용자에게 *지원하고 싶은 회사 하나*를 묻기 (회사명 1개)
2. 회사 카드 생성: `/synapse-ask "회사 카드 <slug>, 제목 '<회사명>' 만들어줘"`
3. 사용자에게 *그 회사가 원하는 기술 스택 1~3개*를 묻고, Obsidian에서 직접 회사 카드의 `positions`에 채우라고 안내
4. 카드가 너무 비어 있으면 결과 품질이 낮음을 *미리* 경고
5. `/synapse-daily` (카드 색인 갱신용)
6. `/synapse-resume <slug>` 호출 → 결과 파일 경로 보고

### 5️⃣ 선택 → 의사결정 체험

1. 사용자에게 *최근 반복적으로 고민한 결정 1개*를 묻기 (예: "프레임워크 A vs B")
2. Profile.md / DecisionPatterns.md가 비어 있을 가능성이 큼 → "처음에는 일반 모드로 답합니다" 안내
3. `/synapse-decide "<그 결정>"` 호출 → 출력에서 `[Profile 없음 - 일반 모드]` 확인
4. "Profile을 채울수록 권장이 *내 패턴*에 맞게 정확해집니다" 설명
5. 다음 단계로: `/synapse-daily` 후 MemoryInbox 검토 흐름을 *짧게* 안내 (실제 검토는 사용자 몫)

### "skip" 선택 → 가벼운 둘러보기

1. `/synapse-daily` 한 번 실행 (시간이 없으면 권유만)
2. 5가지 명령 표 한 번 보여주고 다음 세션에 1번을 다시 호출하라고 안내

## 4단계 — 다음 행동 1개만 남기고 종료

세션을 끝낼 때 *정확히 다음 한 줄*만 강조하세요. 더 많이 알려주면 부담을 줍니다.

```
오늘 한 일: <선택한 답답함의 핵심 1줄>

내일/다음에 와서:
  - 매일 `/synapse-daily` 한 번 (1~3분)
  - 끝나면 `90_System/AI/MemoryInbox/` 새 파일 열어 5분 검토
  - 본격 사용 모드는 `/synapse-assistant` 호출하면 알아서 추천해줍니다
```

종료.

## 절대 하지 말 것

- ❌ 답답함 5개를 한 세션에 다 보여주기
- ❌ `persona update-profile`을 사용자 동의 없이 실행 (비용 발생 + 후보 누적)
- ❌ vault 안 사용자 파일을 임의로 편집 (카드 status 변경, Profile 추가 등은 사용자가 Obsidian에서 직접)
- ❌ `--auto-promote` 같은 자동 승격 모드 사용 (현재 미구현, 의도적 수동)
- ❌ Gatekeeper 우회·보안 설정 변경
